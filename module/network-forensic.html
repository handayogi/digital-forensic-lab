Network Forensics

Apa itu Network Forensics?

Introduction

Network forensic adalah suatu cabang dari digital forensik yang memfokuskan diri pada analisis jaringan digital, termasuk struktur jaringan, protokol, dan metode identifikasi dan ekstraksi informasi dari jaringan digital. Network forensic digunakan untuk merekonstruksi aktivitas pengguna, mengidentifikasi pola perilaku, serta menghubungkan berbagai artefak digital dalam suatu skenario investigasi.

Lesson Overview

  • File memiliki struktur internal (header, body, footer)
  • Ekstensi file tidak selalu mencerminkan format sebenarnya
  • File dapat dimodifikasi atau dimanipulasi
  • File analysis menjadi dasar analisis forensik lanjutan

Learning Outcomes

  • Memahami konsep dasar analisis file digital
  • Mengidentifikasi struktur dan karakteristik file
  • Menentukan peran file sebagai artefak forensik

Practice Scenario

Langkah 1 - Traffic Overview

Memahami struktur umum dari lalu lintas jaringan.

cd evidence/network_forensic
tsark -r {nama_file}.pcap

Fungsi dari perintah atau command tshark -r {nama_file}.pcap:

  • Membaca file PCAP melalui CLI
  • -r, membaca file pcap
  • Menampilkan paket satu per satu

Langkah 2 - Identifikasi Protokol Jaringan

Periksa lalu lintas jaringan untuk mengidentifikasi protokol jaringan apa yang paling dominan.

tshark -r {nama_file}.pcap -q -z io,phs

Fungsi dari perintah atau command tshark -r {nama_file}.pcap -q -z io,phs:

  • Menampilkan statistik distribusi protokol
  • -q, quiet mode
  • -z io,phs, protocol hierarchy statistics

Langkah 3 - Investigasi DNS

Melakukan investigasi terhadap DNS dan mengidentifikasi domain queries yang mencurigakan.

tshark -r {nama_file}.pcap -Y dns -T fields -e dns.qry.name

(Opsional: Menghitung frequensi)

exiftool -r {nama_file}.pcap -Y dns -T fields -e dns.qry.name | sort | uniq -c | sort -nr

Fungsi dari perintah atau command tshark -r {nama_file}.pcap -Y dns -T fields -e dns.qry.name:

  • Mengesktrak nama domain dari lalu lintas DNS
  • -Y dns, filter lalu lintas DNS
  • -T fields, menampilkan output spesifik
  • -e dns.qry.name, menampilkan field domain
  • sort, mengurutkan frekuensi kemunculan domain
  • uniq -c, menghitung frekuensi kemunculan domain
  • sort -nr, mengurutkan kemunculan dari yang terbanyak

Langkah 4 - Analisis Traffic HTTP

Melakukan inspeksi terhadap lalu lintas permintaan protokol HTTP untuk mengidentifikasi aktivitas mencurigakan.

tshark -r {nama_file}.pcap -Y http.request

(Opsional: show request details)

tshark -r {nama_file}.pcap -Y http -V

Fungsi dari perintah atau command tshark -r {nama_file}.pcap -Y http.request:

  • Menampilkan http request

Langkah 5 - Analisis IP Address

Melakukan identifikasi hubungan IP yang mencurigakan.

tshark -r {nama_file}.pcap -T fields -e ip.src -e ip.dst

Fungsi dari perintah atau command tshark -r {nama_file}.pcap -T fields -e ip.src -e ip.dst:

  • Menampilkan pasangan IP dari sumber dan tujuan
