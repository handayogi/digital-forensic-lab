Digital Footprint Investigation

Apa itu Digital Footprint Investigation?

Introduction

Digital footprint investigation atau investigasi jejak digital merupakan proses analisis terhadap jejak digital yang ditinggalkan oleh oleh pengguna atau sistem selama berinteraksi dengan teknologi digital. Jejak ini dapat muncul secara sadar maupun tidak sadar dan tersebar di berbagai sumber digital.

Dalam Digital Forensic, jejak digital digunakan untuk merekonstruksi aktivitas pengguna, mengidentifikasi pola perilaku, serta menghubungkan berbagai artefak digital dalam suatu skenario investigasi. Analisis digital foorprint dalam modul ini mencakup:

  • Aktivitas pengguna pada sistem
  • Riwayat aktivitas pada browser atau aplikasi
  • Log sistem dan file cache
  • Artefak aktivitas jaringan (sederhana)

Jejak digital dianalisis secara korelatif untuk membangun kronologi aktivitas.

Lesson Overview

  • Jejak digital berasal dari berbagai sumber (browser atau aplikasi)
  • Digital footprint investigation merupakan proses analisis terhadap jejak digital yang ditinggalkan.
  • Jejak digital dapat muncul secara sadar maupun tidak sadar
  • Analisis jejak digital mendukung rekonstruksi aktivitas

Learning Outcomes

  • Memahami konsep dasar investigasi jejak digital
  • Mengidentifikasi sumber jejak digital
  • Menganalisis jejak aktivitas digital berdasarkan artefak yang tertinggal atau tersedia

Practice Scenario

Langkah 1 - Log Overview

Memahami log file yang tersedia

cd evidence/digital_footprint
ls -lah

Fungsi dari perintah atau command ls -lah:

  • l, menampilkan detail file (permission, owner, size, timestamp).
  • a, menampilkan semua file, termasuk file yang tersembunyi.
  • h, menampilkan file dalam format human-readable.

Langkah 2 - Analisis Log Otentikasi

Melakukan inspeksi terhadap log otentikasi untuk mengidentifikasi aktivitas percobaan login yang mencurigakan.

grep "Failed password" auth.log
grep "Accepted password" auth.log

Fungsi dari perintah atau command grep "Failed password" auth.log dan grep "Accepted password" auth.log:

  • Mencari kata kunci "Failed password" dan "Accepted password" dalam file log

Langkah 3 - Deteksi Privilege Escalation

Identifikasi penggunaan command administrative.

grep "sudo" auth.log

Fungsi dari perintah atau command grep "sudo" auth.log:

  • Mencari kata kunci "sudo" dalam file log untuk mendeteksi penggunaan hak administrative

Langkah 4 - Analisis Histori Perintah User

Rekonstruksi perilaku user.

cat bash_history.txt

Fungsi dari perintah atau command cat bash_history.txt:

  • Menampilkan isi file riwayat perintah user

Langkah 5 - Timeline Reconstruction

Hubungkan semua artefak yang ditemukan ke timeline

less auth.log

Fungsi dari perintah atau command less auth.log:

  • Menampilkan log secara interaktif
